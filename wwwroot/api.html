Authentication

All API requests must include a Bearer token in the Authorization header. You can generate and manage tokens from the dashboard under Settings → API Access. Tokens don't expire by default but can be revoked at any time.

Keep your token out of shell history, version control, and anywhere else you'd be embarrassed to have it discovered. It provides full read/write access to your DNS filter configuration.

Authorization header
Authorization: Bearer YOUR_API_TOKEN
Example curl
curl -H "Authorization: Bearer YOUR_TOKEN" \ http://localhost:8080/api/stats
Requests without a valid token return 401 Unauthorized. Requests with a revoked token return the same — no hints about whether the token ever existed.
Error Handling

All errors return a JSON object with a code and a human-readable message. The HTTP status code matches the error type.

Error response shape
{ "code": "NOT_FOUND", "message": "domain not found in whitelist" }
HTTP StatusCodeMeaning
400BAD_REQUESTMissing or invalid request body
401UNAUTHORIZEDMissing or invalid token
404NOT_FOUNDResource doesn't exist
409CONFLICTResource already exists (e.g. domain already whitelisted)
500INTERNAL_ERRORSomething went wrong — check service logs
GET /api/stats Filtering statistics summary

Returns aggregate statistics about DNS Filter's activity: total queries processed, total blocked, breakdown by category, and current uptime. This is the same data shown on the dashboard home screen.

No parameters required. Just hit it and you'll get the full current state.

Request
curl -H "Authorization: Bearer TOKEN" \ http://localhost:8080/api/stats
Response 200
{ "total_queries": 48291, "total_blocked": 3847, "block_rate_pct": 7.96, "blocked_by_category": { "adult": 2103, "malware": 891, "gambling": 853 }, "uptime_seconds": 86400, "started_at": "2026-03-17T00:00:00Z" }
GET /api/blocked Recently blocked queries

Returns a paginated list of the most recently blocked DNS queries. Useful for reviewing what's being caught, debugging false positives, or just satisfying your curiosity about what your household devices are trying to reach.

Query ParamTypeRequiredDescription
limitintegeroptionalNumber of results to return. Default: 50. Max: 500.
offsetintegeroptionalPagination offset. Default: 0.
categorystringoptionalFilter by category (e.g. "malware", "adult").
Request
curl -H "Authorization: Bearer TOKEN" \ "http://localhost:8080/api/blocked?limit=20&category=malware"
Response 200
{ "total": 891, "results": [ { "domain": "evil-tracker.net", "category": "malware", "timestamp": "2026-03-17T23:14:02Z", "client_ip": "192.168.1.42" } ] }
GET/api/whitelistList whitelisted domains

Returns all domains currently on the whitelist. These domains are always allowed through, regardless of blocklist membership.

Request
curl -H "Authorization: Bearer TOKEN" \ http://localhost:8080/api/whitelist
POST/api/whitelistAdd domain to whitelist

Permanently allow a domain, even if it appears on an active blocklist. Takes effect immediately — no restart required. Returns 409 if the domain is already whitelisted.

Body FieldTypeRequiredDescription
domainstringrequiredThe domain to whitelist (e.g. "example.com"). Subdomains are not automatically included.
Request
curl -X POST http://localhost:8080/api/whitelist \ -H "Authorization: Bearer TOKEN" \ -H "Content-Type: application/json" \ -d '{"domain": "example.com"}'
DELETE/api/whitelist/:domainRemove domain from whitelist

Remove a domain from the whitelist. It will once again be subject to blocklist filtering. Returns 404 if the domain isn't on the whitelist.

Request
curl -X DELETE \ -H "Authorization: Bearer TOKEN" \ http://localhost:8080/api/whitelist/example.com
GET/api/blocklistsList all blocklists

Returns all configured blocklists — built-in category lists and any custom ones you've added — along with their status, URL, domain count, and last updated time.

Response 200
[ { "id": 1, "name": "Adult Content", "url": "builtin://adult", "enabled": true, "domain_count": 852044, "last_updated": "2026-03-15T12:00:00Z" } ]
POST/api/blocklistsAdd a custom blocklist

Add an external blocklist by URL. DNS Filter will fetch, parse, and load the list immediately. Supports standard hosts-format and plain domain-per-line formats.

Body FieldTypeRequiredDescription
urlstringrequiredURL to the blocklist file. Must be publicly accessible.
namestringoptionalDisplay name for the list. Defaults to the URL hostname.
enabledbooleanoptionalWhether to activate the list immediately. Default: true.
Request
curl -X POST http://localhost:8080/api/blocklists \ -H "Authorization: Bearer TOKEN" \ -H "Content-Type: application/json" \ -d '{"url":"https://example.com/blocklist.txt","name":"My Custom List","enabled":true}'
DELETE/api/blocklists/:idRemove a blocklist

Remove a custom blocklist by its numeric ID. Built-in category lists cannot be deleted (only disabled). Get the list ID from GET /api/blocklists.

Request
curl -X DELETE \ -H "Authorization: Bearer TOKEN" \ http://localhost:8080/api/blocklists/5
