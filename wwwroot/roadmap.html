v1.0
THE FOUNDATION

The first stable release. Everything that makes DNS Filter actually work. It does the job and gets out of the way.

High-performance DNS server in Go
The core engine. A real DNS server, not a bash script with pipe dreams. Handles thousands of concurrent queries, forwards allowed domains upstream, returns NXDOMAIN or 0.0.0.0 for blocked ones. Built to run quietly forever without consuming resources.
Done
Category-based blocklists with curated content
Adult, malware, gambling, and phishing categories — each backed by community-maintained domain lists with hundreds of thousands of entries. Toggle entire categories on or off. No manual domain management required for the basics.
Done
Web management dashboard
A full management interface at localhost:8080. Real-time stats, blocklist management, whitelist control, query logs. Fast because it's served locally, not from someone's CDN. Change the default password. Please. We cannot stress this enough.
Done
Full REST API with token authentication
Complete programmatic control. Every feature available in the dashboard is also available through the API. Integrate DNS Filter into homelab automation, scripts, or monitoring pipelines. Token-based auth, consistent response shapes, proper HTTP status codes.
Done
SQLite storage — zero external dependencies
All logs, configuration, blocklists, and whitelist entries stored in a local SQLite database. No PostgreSQL server to maintain, no Redis to monitor, no Docker-compose file with 14 services just to run a DNS filter. One binary, one database file.
Done
Cross-platform service support (Linux / macOS / Windows)
Ships with service files for systemd (Linux), launchd (macOS), and Windows Service Manager. Install once, starts on every boot. Survives reboots, survives your cron job wars, survives most things.
Done
v1.1
GOING ENCRYPTED

Modern DNS is moving toward encryption. v1.1 brings DoH and DoT support so DNS Filter keeps pace. Also: better analytics, because knowing is half the battle.

DNS-over-HTTPS (DoH) support
Modern browsers and clients increasingly use DNS-over-HTTPS, bypassing traditional DNS resolvers entirely. v1.1 adds a DoH endpoint so DNS Filter can intercept and filter these queries too. No more sneaky browser-level bypasses. We see you, Firefox.
In Progress
DNS-over-TLS (DoT) support
Encrypted DNS queries over TLS (port 853). Useful for privacy-focused deployments where you want DNS traffic encrypted in transit — even to your own resolver. Particularly valuable when running DNS Filter on a remote server or shared network.
In Progress
Advanced analytics dashboard
The current stats screen shows totals. v1.1 brings charts, trends, time-series query graphs, top 10 blocked domains, top 10 clients, block rate over time, and CSV export. For those of us who love a good graph and spend 45 minutes tuning the visualisation instead of doing actual work.
In Progress
v2.0+
THE BIG STUFF

Longer-term ideas. Some of these are well-defined, some are still "wouldn't it be cool if...". All are open for community input and PRs.

Machine learning-based domain classification
Static blocklists will always lag behind new bad domains. ML-based classification can identify suspicious domains based on patterns — newly registered domains, typosquatting patterns, DGA (domain generation algorithm) detection — even before they appear on any public list.
Planned
Mobile app for remote management
iOS and Android app that connects to your DNS Filter instance and lets you manage it from anywhere. View stats on your phone, add whitelist exceptions on the fly, check what's been blocked while you're out. Useful for families and small offices where the admin isn't always at a desk.
Planned
Per-device and multi-user profiles
Different filtering rules for different devices or users. Strict rules for the kids' devices, looser rules for the adults, custom rules for that one family member who insists on watching things they probably shouldn't. Each profile gets its own blocklist config, schedule, and whitelist.
Planned
Cloud sync for configuration
Optionally sync your config, whitelists, and custom blocklists across multiple DNS Filter instances. Run it on your home server and your office router and keep them in sync. Fully opt-in — the default is and always will be fully local.
Planned
Pre-built binaries and package manager support
Right now you build from source. We want to ship pre-built binaries for Linux (x86_64, ARM64), macOS, and Windows so Go doesn't need to be installed at all. Plus Homebrew tap, apt/deb package, and AUR for Arch users (they'll add it themselves anyway).
Planned
