What it does

FEATURES
& CAPABILITIES

DNS Filter ships with everything you need to run serious network-level content filtering. No plugins, no paid tiers, no "pro plan to unlock the good stuff."

High-Performance DNS Server

Written from scratch in Go — not a wrapper around something else, not a Python script with ambitions. A real DNS server that handles thousands of queries per second with sub-millisecond overhead. It runs quietly in the background and won't eat your RAM like a browser with 40 tabs open (we're not naming names).

Upstream DNS forwarding is configurable. Use Google (8.8.8.8), Cloudflare (1.1.1.1), your ISP's resolver, or your own local unbound setup. DNS Filter doesn't care — it just checks and forwards.

DNS-Level Content Filtering

Blocking at the DNS level is fundamentally different from browser-based filtering. It doesn't matter which browser you use, which app makes the request, or whether the user knows how to open incognito mode. If a domain is on the blocklist, the query returns 0.0.0.0. The request never leaves your machine.

This works for every device you route through it — phones, laptops, smart TVs, that IoT toaster that for some reason needs internet access.

Category-Based Blocking

Don't want to manage a list of a million domains by hand? Good, because that would be terrible. DNS Filter organizes blocked content into categories — adult, malware, gambling, phishing, ads, and more — each backed by maintained community blocklists.

Enable or disable entire categories with a single config line or a dashboard toggle. When a new malware domain shows up, the list updates and you don't have to do anything. That's the deal.

Web Management Dashboard

After setup, you never have to touch the terminal again if you don't want to. The web dashboard at localhost:8080 gives you full control over everything: blocklists, whitelist, category toggles, schedule rules, and live query logs.

It's clean, it's fast, and it loads instantly because it's served from your own machine — not from a CDN in Virginia with a four-second first contentful paint. Default login is admin / changeme. Change that. Immediately.

Analytics & Query Logging

Every DNS query — allowed or blocked — is logged to SQLite with a timestamp, domain, result, category, and source. You can filter, search, and export. Want to know what your smart TV has been up to at 3am? DNS Filter will tell you. (You might not like the answer.)

The dashboard shows real-time stats: total queries, block rate, top blocked domains, and query trends over time. All stored locally, none of it going anywhere.

Schedule-Based Filtering

Different rules for different times of day. Block gaming and social media sites during work hours (9–5, Monday–Friday, ruthlessly). Lift restrictions in the evenings. Apply stricter filtering overnight. Configure it once and let the scheduler handle the rest.

Useful for shared household setups, managed devices, school environments, or anyone who knows that they personally cannot be trusted with unrestricted internet access after midnight.

Whitelist Management

Sometimes a domain you actually need ends up on a blocklist — it happens. The whitelist system lets you permanently allow specific domains regardless of what category or list they're on. Manage it through the dashboard, the config file, or the API. Changes take effect immediately without restarting anything.

You can also add custom block entries for domains that aren't on any public list. Running a workplace device and want to block your employees' favourite time-wasting site? Add it. No judgment. (A little judgment.)

Whole-Network or Single-Device

Run DNS Filter on a server or Raspberry Pi, point your router's DNS at it, and protect your entire home or office network in one move. Every device — phones, tablets, smart TVs, the kids' laptops — all filtered, no per-device configuration required.

Or run it locally on a single machine and point just that machine's DNS resolver to 127.0.0.1. Both modes work identically. The only difference is the scale of surveillance you're comfortable with.

Configuration

ONE YAML FILE.
THAT'S IT.

The entire DNS Filter configuration lives in a single configs/config.yaml file. No registry keys, no hidden dotfiles, no environment variable scavenger hunt. Open it, edit it, restart the service.

You can set upstream DNS resolvers, configure which port the DNS server listens on, choose which filter categories are active, set the database path, and configure the API server — all in one place, with comments that actually explain what things do. Revolutionary, we know.

The config file is also your source of truth. If you ever lose access to the dashboard or something breaks, you can restore your exact setup from this file alone. Back it up alongside your blocklists and you're fully recoverable.

  • Configurable DNS port (default: 53)
  • Configurable API/Dashboard port (default: 8080)
  • Upstream DNS resolver (any resolver works)
  • Per-category enable / disable
  • Custom database file path
  • Hot-reload on most changes
configs/config.yaml ● valid
# DNS Filter configuration # Edit this file and restart the service server: dns_port: 53 # needs root/admin api_port: 8080 # dashboard + REST API upstream_dns: "8.8.8.8:53" filtering: enabled: true block_categories: - adult # 850k+ domains - malware # constantly updated - gambling # your wallet thanks you database: path: "./data/dns-filter.db" schedule: enabled: false # configure rules in the dashboard
