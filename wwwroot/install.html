Prerequisites

Before you start, make sure you have these. None of them are unusual — if you're running a modern Linux system you probably already have most of this.

Go 1.21 or higher
Required to build from source. Not needed if using the pre-built binary (coming soon).
Root / Admin privileges
DNS runs on port 53. Port 53 requires root on Linux/macOS. On Windows, run as Administrator.
git (optional)
Used by the setup script to clone the repo. Falls back to tarball download if git isn't available.
DNS Filter listens on port 53. If you have another DNS service running (like systemd-resolved or dnsmasq), you may need to disable it first to avoid a port conflict. We'll cover this in the service setup section.
Quick Install

The fastest path. One curl command clones the repo, builds the binary, installs the systemd service, and starts everything. Requires Git and Go on your machine.

Important: Always review a shell script before piping it to sudo bash. We're not saying don't do it — we're saying know what you're running. The script is at setup.sh and is fully readable on GitHub.
1
Run the one-liner

This downloads, builds, and installs everything. Grab a coffee — the Go build takes about 30 seconds.

bash
curl -fsSL https://raw.githubusercontent.com/RDXFGXY1/dnsFilterApp/main/setup.sh | sudo bash
2
Verify the service is running
bash
sudo systemctl status dns-filter
You should see "Active: active (running)". If not, check the service logs: journalctl -u dns-filter -n 50
3
Test DNS filtering

Point a dig query at localhost to confirm it's responding.

bash
# This should resolve normally dig google.com @127.0.0.1 # A blocked domain should return 0.0.0.0 dig blocked-example.com @127.0.0.1
4
Open the dashboard

Navigate to the web interface in your browser. Default credentials are admin / changeme — please, please change the password.

browser
http://localhost:8080
Manual Build

For those who want to see exactly what's happening, or who want to customize the build. Clone, build, run. No magic.

1
Clone the repository
git
git clone https://github.com/RDXFGXY1/dnsFilterApp.git cd dnsFilterApp
2
Install dependencies
go
go mod download
3
Build the binary
make
# Build for your current platform make build # Or build for all platforms at once make build-all # Or go directly without make go build -o build/dns-filter ./cmd/server
4
Run the installer
bash
sudo bash scripts/install.sh
5
Development mode (no install required)

Want to hack on it without installing? Run directly with auto-reload.

dev
go run cmd/server/main.go --dev
In dev mode, port 53 may still require sudo. Use sudo -E go run cmd/server/main.go --dev if you see a permission error.
Run as a Service

DNS Filter is designed to run as a background system service that starts automatically on boot. Here's how to set that up on each platform.

1
Install the service file
systemd
sudo cp scripts/dns-filter.service /etc/systemd/system/ sudo systemctl daemon-reload
2
Enable & start
bash
sudo systemctl enable dns-filter # start on boot sudo systemctl start dns-filter # start now sudo systemctl status dns-filter # verify
3
Handle systemd-resolved conflict (if needed)

On modern Ubuntu/Debian, systemd-resolved may already be using port 53. Disable it first:

bash
sudo systemctl disable systemd-resolved sudo systemctl stop systemd-resolved # Then recreate resolv.conf pointing to dns-filter sudo rm /etc/resolv.conf sudo bash -c 'echo "nameserver 127.0.0.1" > /etc/resolv.conf'
Disabling systemd-resolved on Ubuntu can break DNS until dns-filter is running. Do this step last, right before starting the service.
1
Open PowerShell as Administrator

Right-click Start → Windows PowerShell (Admin). DNS requires admin on Windows too.

2
Install and start the service
powershell
.\dns-filter.exe install .\dns-filter.exe start
3
Verify via Services Manager

Open services.msc and look for "DNS Filter". It should show as Running.

1
Install the launchd plist
launchd
sudo cp scripts/com.dnsfilter.plist /Library/LaunchDaemons/ sudo launchctl load /Library/LaunchDaemons/com.dnsfilter.plist
2
Verify it loaded
bash
sudo launchctl list | grep dnsfilter
Set System DNS

DNS Filter needs to be your system's DNS resolver to intercept queries. Point your OS at 127.0.0.1 and every DNS lookup goes through it.

bash
# Direct resolv.conf edit (simplest) sudo rm -f /etc/resolv.conf sudo bash -c 'echo "nameserver 127.0.0.1" > /etc/resolv.conf' # Lock it so NetworkManager doesn't overwrite it sudo chattr +i /etc/resolv.conf # Test it dig google.com
To undo the immutable flag later: sudo chattr -i /etc/resolv.conf
1
Control Panel → Network and Internet → Network Connections
2
Right-click your active connection → Properties
3
Select "Internet Protocol Version 4 (TCP/IPv4)" → Properties
4
Set Preferred DNS to 127.0.0.1

Leave Alternate DNS blank or set it to 8.8.8.8 as a fallback (for when dns-filter isn't running).

1
System Preferences → Network
2
Select your active connection → Advanced → DNS tab
3
Click + and add 127.0.0.1 as the first DNS server
Config File

Everything lives in configs/config.yaml. Edit it, save it, restart the service. That's the whole workflow.

configs/config.yaml
# ── DNS Filter Configuration ────────────────────────────── server: dns_port: 53 # needs root privileges on Linux/macOS api_port: 8080 # dashboard and REST API port upstream_dns: "8.8.8.8:53" # where to forward allowed queries filtering: enabled: true block_categories: - adult # explicit content - malware # malware C&C, phishing - gambling # gambling sites # - ads # ad networks (optional) # - tracking # tracker domains (optional) database: path: "./data/dns-filter.db" schedule: enabled: false # rules are configured through the web dashboard
After editing config.yaml, restart the service to apply changes: sudo systemctl restart dns-filter
Dashboard Setup

The web dashboard runs at http://localhost:8080 and gives you full control without touching the terminal.

1
Open the dashboard
browser
http://localhost:8080
2
Log in and immediately change the password

Default credentials are admin / changeme. Go to Settings → Change Password. This is not optional. The dashboard is on your local network and accessible to anyone on it.

3
Check your stats

The home screen shows total queries, block rate, and top blocked domains. If you see queries coming in, DNS Filter is working correctly. If nothing shows up, your system DNS hasn't been pointed to 127.0.0.1 yet.

Custom Blocklists

The built-in categories cover most use cases, but you can add any external blocklist URL or define your own entries via the CLI or API.

1
Add a blocklist via CLI
dns-filter cli
./dns-filter blocklist add --url "https://example.com/blocklist.txt"
2
Add a blocklist via API
REST API
curl -X POST http://localhost:8080/api/blocklists \ -H "Authorization: Bearer YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{"url": "https://example.com/blocklist.txt", "enabled": true}'
3
Or manage via the dashboard

Go to Dashboard → Blocklists → Add New. Paste the URL, click Add, and it's immediately active. You can toggle, remove, or force-refresh any list from the same screen.
